# 小小菜单 - 项目使用指南

## 项目简介

这是一个基于 React + TypeScript 的点菜系统原型，支持两种用户角色：客户和厨师。

## 功能特性

### 1. 登录系统
- 支持两种身份登录：
  - **客户身份**：浏览和选择喜欢的菜品
  - **厨师身份**：查看客户订单并确认制作

### 2. 客户功能
- 查看 AI 每日推荐的菜品
- 从推荐菜品中选择多个喜欢的菜
- 查看菜品详细信息（描述、难度、烹饪时间）
- 实时保存选择状态

### 3. 厨师功能
- 查看客户选择的所有菜品
- 从客户选择中确认要制作的菜品
- 查看完整菜谱（食材清单、制作步骤）
- 实时保存确认状态

## 技术栈

- React 19.2.0
- TypeScript 4.9.5
- React Router DOM 7.9.5
- Create React App 5.0.1

## 如何使用

### 启动项目

项目已自动启动，访问地址：
- **本地访问**: http://localhost:3000
- **局域网访问**: http://192.168.110.25:3000

如需重新启动：
```bash
cd /Users/hufangwei/tiny-menu-web/tiny-menu-web
npm start
```

### 使用流程

#### 作为客户使用：
1. 打开浏览器访问 http://localhost:3000
2. 在登录页面选择"客户"身份
3. 点击登录按钮
4. 浏览 AI 推荐的 6 道菜品
5. 点击菜品卡片进行选择（已选择的菜品会显示绿色边框和勾选标记）
6. 可以随时取消选择

#### 作为厨师使用：
1. 打开浏览器访问 http://localhost:3000
2. 在登录页面选择"厨师"身份
3. 点击登录按钮
4. 查看客户已选择的菜品
5. 点击"查看菜谱"按钮查看详细制作方法
6. 点击"确认制作"按钮确认要做的菜（已确认的菜品会显示蓝色边框）
7. 可以随时取消确认

### 测试建议

1. 打开两个浏览器标签页
2. 一个标签页使用客户身份，另一个使用厨师身份
3. 在客户页面选择几道菜
4. 刷新厨师页面，查看客户的选择
5. 在厨师页面确认要制作的菜品

## Mock 数据说明

### 用户数据
- 客户：女朋友
- 厨师：厨师

### 菜品数据（共6道）
1. 宫保鸡丁 - 中等难度，25分钟
2. 番茄炒蛋 - 简单，10分钟
3. 红烧肉 - 困难，90分钟
4. 清蒸鲈鱼 - 简单，20分钟
5. 麻婆豆腐 - 中等难度，15分钟
6. 糖醋里脊 - 中等难度，30分钟

每道菜都包含：
- 菜品名称和描述
- 图片
- 食材清单
- 详细制作步骤
- 烹饪时间
- 难度等级

## 数据存储

项目使用 localStorage 存储数据：
- `currentUser`: 当前登录用户信息
- `customerSelections`: 客户选择的菜品
- `chefConfirmations`: 厨师确认的菜品

## 页面路由

- `/` - 登录页面
- `/customer` - 客户选菜页面
- `/chef` - 厨师确认页面

## 样式特点

- 响应式设计，支持移动端和桌面端
- 渐变色背景
- 卡片式布局
- 流畅的动画效果

- 这是一个原型项目，使用 Mock 数据
- 数据存储在浏览器本地，清除浏览器数据会丢失
- 不同浏览器之间数据不共享
- 建议使用最新版本的 Chrome、Firefox 或 Safari 浏览器
